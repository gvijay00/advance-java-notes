## 1. What is JSP?

* **JSP (JavaServer Pages)** is a server-side technology that allows developers to create **dynamic web content** using **HTML + Java code**.
* It is built on top of **Servlets** (JSP is internally converted into a Servlet by the web container).
* JSP simplifies coding by allowing HTML with embedded Java, unlike Servlets where everything must be written inside Java code.

âœ… **Key Point**: JSP = HTML + Java (server-side).

---

## 2. JSP Life Cycle

Every JSP page goes through several phases when deployed:

1. **Translation Phase** â†’ JSP file (`.jsp`) is translated into a Java Servlet.
2. **Compilation Phase** â†’ The generated Servlet `.java` is compiled into `.class`.
3. **Loading & Initialization** â†’ Servlet class is loaded and `jspInit()` method is called.
4. **Request Processing** â†’ Each client request calls `_jspService()` method.
5. **Destroy Phase** â†’ When server shuts down or page is removed, `jspDestroy()` is called.

## 3. JSP Architecture

* **Client (Browser)** sends request for `.jsp`.
* **Web Server (Tomcat/Jetty/GlassFish)** forwards request to JSP Engine.
* JSP Engine translates `.jsp` â†’ Servlet â†’ executes Java code â†’ generates **HTML Response**.
* **Client receives pure HTML output** (never sees Java code).

## 4. JSP Syntax Elements

JSP provides different **elements** to embed Java code into HTML:

### a) Directives

Provide instructions to the JSP container.
Syntax: `<%@ directive attribute="value" %>`

* **Page Directive** â†’ `<%@ page language="java" contentType="text/html" %>`
* **Include Directive** â†’ `<%@ include file="header.jsp" %>`
* **Taglib Directive** â†’ `<%@ taglib uri="..." prefix="..." %>`

### b) Scriptlets

Embed Java code inside HTML.

```jsp
<%
   int a = 10;
   out.println("Value of a: " + a);
%>
```

### c) Expressions

Used to output values directly.

```jsp
<%= new java.util.Date() %>
```

### d) Declarations

Define methods/variables inside JSP.

```jsp
<%! int count = 0; %>
<%! public int getCount() { return ++count; } %>
```

### e) Comments

```jsp
<%-- This is a JSP comment --%>
```

## 5. JSP Implicit Objects

JSP provides **9 built-in objects** (no need to declare).

| Object      | Type                | Description        |
| ----------- | ------------------- | ------------------ |
| request     | HttpServletRequest  | Request data       |
| response    | HttpServletResponse | Response data      |
| out         | JspWriter           | Output to client   |
| session     | HttpSession         | Session tracking   |
| application | ServletContext      | Application scope  |
| config      | ServletConfig       | Config params      |
| pageContext | PageContext         | Page info          |
| page        | Object (this)       | Current JSP page   |
| exception   | Throwable           | Exception handling |

## 6. JSP Action Tags

Used for **dynamic operations** (XML-style tags):

* `<jsp:include page="header.jsp" />` â†’ includes another page.
* `<jsp:forward page="next.jsp" />` â†’ forwards request to another JSP/Servlet.
* `<jsp:useBean id="u" class="com.model.User" />` â†’ create JavaBean.
* `<jsp:setProperty name="u" property="*" />` â†’ set bean property.
* `<jsp:getProperty name="u" property="name" />` â†’ get bean property.

## 7. Example JSP Page

**hello.jsp**

```jsp
<%@ page contentType="text/html" language="java" %>
<html>
<head><title>Hello JSP</title></head>
<body>
   <h2>Welcome to JSP!</h2>
   Current Time: <%= new java.util.Date() %> <br>

   <%
       String name = request.getParameter("user");
       if(name != null) {
           out.println("Hello, " + name + "!");
       } else {
           out.println("Hello, Guest!");
       }
   %>
</body>
</html>
```

## 8. Advantages of JSP over Servlet

| Servlet                            | JSP                      |
| ---------------------------------- | ------------------------ |
| 100% Java code                     | Mix of HTML + Java       |
| Harder to write presentation logic | Easier for web designers |
| Good for business logic            | Good for UI (View)       |
| Must write out.println() for HTML  | Direct HTML inside page  |
